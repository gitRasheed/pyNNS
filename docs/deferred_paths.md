# Deferred Paths

Deferred paths include explicit `NotImplementedError` guards plus public API
surfaces that remain blocked by those guarded paths. Keep the dependency notes
current so temporary scope limits stay visible.

| Area | Deferred path | Reason | Dependency / next action |
|---|---|---|---|
| Boost | `threshold` on the `n_features > 10` stochastic path | The ordinary stochastic epoch keeper path, including `ts_test`, is supported structurally with PyNNS RNG. Installed R errors for `threshold=` because `test.features` is never built. | Keep threshold guarded to match installed-R error behavior |
| Boost | named-data-frame factor predictor ordering | Explicit-level factor predictors, including multiple factor columns, are supported with PyNNS positional `X1`, `X2`, ... semantics. Installed R data frames with semantic names sort columns alphabetically before `data.matrix`, which changes feature diagnostics and predictions. | Keep PyNNS positional; add a named-column API only if a future public data-frame boundary is introduced |
| Differentiation | multivariate `dy.d_` vectorized `wrt` | `dy_dx(..., eval_point="overall")` and numeric univariate `dy_dx` evaluation points are supported. `dy_d` with a single `wrt` column (scalar wrt) is now ported with R-matching finite-difference grids. Vectorized `wrt` (passing multiple regressors in one call) is not yet ported; call `dy_d` once per regressor. | Port multi-wrt vectorized dispatch |
| Stack | mixed factor predictor stacked method `(1, 2)` diagnostics | `nns_stack` methods 1 and 2 support explicit `factor_levels`, and pure factor-predictor `(1, 2)` matches installed R's fallback to method 1. The remaining mixed factor/numeric stacked `(1, 2)` path is rejected because installed-R probes show divergent stacked method-1 internals after method-2 projection. | Map R's stacked method-1 design reconstruction and diagnostic field assignment before broadening mixed factor predictor parity claims |
| Multivariate regression | direct `factor_2_dummy=True` raw predictor path | Public `nns_reg` factor predictors are supported, but installed R direct `NNS.M.reg` raw factor input errors; PyNNS keeps direct `nns_m_reg` raw factor input rejected unless another public R path requires it | Keep rejected; revisit only if a public installed-R path depends on direct `NNS.M.reg` factor expansion |
| VAR | default VAR path | `NNS.ARMA.optim` and `nns_stack(ts_test)` are ported, but installed `NNS.VAR` feeds named lagged data frames into `NNS.stack`; PyNNS currently matches the positional matrix stack semantics, not the named-data-frame diagnostics/predictions that VAR uses | Map R named-data-frame stack semantics, then port `NNS.VAR` |
| Nowcast | nowcast wrapper | Installed R fetches external macroeconomic series and delegates to `NNS.VAR`; PyNNS cannot faithfully expose the wrapper until `nns_var` is real, and it should avoid implicit runtime data-provider side effects until that API is designed | Port `NNS.VAR`, then design an explicit data-ingestion boundary for `nns_nowcast` |
