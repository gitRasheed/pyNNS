# Deferred Paths

Deferred paths include explicit `NotImplementedError` guards plus public API
surfaces that remain blocked by those guarded paths. Keep the dependency notes
current so temporary scope limits stay visible.

| Area | Deferred path | Reason | Dependency / next action |
|---|---|---|---|
| Boost | `threshold` on the `n_features > 10` stochastic path | The ordinary stochastic epoch keeper path, including `ts_test`, is supported structurally with PyNNS RNG. Installed R errors for `threshold=` because `test.features` is never built. | Keep threshold guarded to match installed-R error behavior |
| Boost | named-data-frame factor predictor ordering | Explicit-level factor predictors, including multiple factor columns, are supported with PyNNS positional `X1`, `X2`, ... semantics. Installed R data frames with semantic names sort columns alphabetically before `data.matrix`, which changes feature diagnostics and predictions. | Keep PyNNS positional; add a named-column API only if a future public data-frame boundary is introduced |
| Differentiation | `dy_d` vectorized `wrt` with `eval_points` other than `"mean"` | `dy_dx(..., eval_point="overall")` and numeric univariate `dy_dx` evaluation points are supported. `dy_d` has enforced scalar parity for `eval_points="mean"` and `"median"`, and vectorized `wrt` parity for `eval_points="mean"` with `mixed=False`. Other vectorized `wrt` evaluation modes remain deferred. | Keep vectorized non-mean modes guarded to `NotImplementedError` |
| Differentiation | `dy_d` scalar `eval_points="last"` parity review | Scalar `last` returns the expected public structure but remains boundary/extrapolation-sensitive. Focused parity shows a material first-derivative difference on the linear boundary case, so this is not yet an enforced parity mode. | Keep an xfail parity test until the boundary-mode derivative is aligned or intentionally documented as divergent |
| Differentiation | `dy_d` scalar `eval_points="obs"` / `"apd"` parity gap | Scalar distribution-mode outputs have the correct public structure and use cumulative bandwidth perturbation. PyNNS now mirrors R's public multivariate `NNS.copula(cbind(wrt, wrt, y))` relevance scalar in this branch, but focused parity still shows material numeric differences from installed R, especially second derivatives. | Keep xfail parity tests for obs/apd second-derivative gaps before deciding whether to chase R's implementation-specific distribution aggregation |
| Multivariate regression | direct `factor_2_dummy=True` raw predictor path | Public `nns_reg` factor predictors are supported, but installed R direct `NNS.M.reg` raw factor input errors; PyNNS keeps direct `nns_m_reg` raw factor input rejected unless another public R path requires it | Keep rejected; revisit only if a public installed-R path depends on direct `NNS.M.reg` factor expansion |
| Nowcast | live/default `nns_nowcast` provider | Public `nns_var` is implemented for numeric matrix-like inputs with `dim_red_method="cor"`, `dim_red_method="NNS.dep"`, `dim_red_method="NNS.caus"`, and `dim_red_method="all"`. Deterministic user-supplied panel nowcasting is implemented as `nns_nowcast_panel`. Explicit provider calls are supported through `nns_nowcast(fetch=True, provider_backend=...)`, including local CSV panels via `CsvNowcastProvider`. Live/default provider fetching remains deferred. Installed R fetches external macroeconomic series and delegates to `NNS.VAR`; PyNNS should avoid implicit runtime data-provider side effects. | Keep default live fetching guarded; add live FRED/Yahoo providers only behind an explicit optional provider boundary |
